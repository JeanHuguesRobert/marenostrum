# AGENTS.md — Mare Nostrum

## Status

This file is the local operational mandate for AI agents working in this repository.

It is not the doctrine itself.  
It is not the corpus itself.  
It is a governed operational projection of the corpus for this repository.

The source methodological references are:

- [`cogentia/research/agent_configuration_layer.md`](https://github.com/JeanHuguesRobert/cogentia/blob/main/research/agent_configuration_layer.md)
- [`cogentia/research/optimistic_mainline_governance.md`](https://github.com/JeanHuguesRobert/cogentia/blob/main/research/optimistic_mainline_governance.md)

## Repository role

`marenostrum` is the repository for the Mare Nostrum layer of the ecosystem.

Its purpose is to explore, structure, document, and progressively stabilize a Mediterranean framework for:

- territorial capability;
- island cooperation;
- energy sovereignty;
- food sovereignty;
- digital and inferential sovereignty;
- anti-capture institutional design;
- open, non-profit, or consortium-like governance;
- long-term cooperation between Corsica, Mediterranean islands, and Mediterranean shores.

This repository is strategic, but it must remain rigorous, traceable, and non-declamatory.

## Core distinction

Before editing or creating any document, distinguish clearly between:

- fact;
- hypothesis;
- scenario;
- doctrine;
- public formulation;
- technical architecture;
- governance model;
- political interpretation;
- economic model;
- legal or institutional claim;
- source document;
- derived publication;
- temporary trace;
- continuation to explore.

Do not present hypotheses as facts.  
Do not present exploratory scenarios as adopted doctrine.  
Do not transform political intuition into stabilized institutional language without explicit human validation.

## Corpus hierarchy

Use the following hierarchy when stabilizing content:

1. existing source documents in this repository;
2. source documents in `cogentia`, when the issue concerns method, pipeline, agents, corpus, continuations, or traceability;
3. source documents in `FractaVolta`, when the issue concerns energy, compute, packets, gateways, technical architecture, or operational implementation;
4. source documents in `barons-Mariani`, when the issue concerns patrimony, doctrine, long-form public reasoning, or institutional continuity;
5. conversation material, only as temporary material unless explicitly stabilized;
6. external sources, only when cited, dated, and evaluated.

If a claim is not grounded, mark it as:

```text
to verify
```

or:

```text
hypothesis
```

## Direct-main rule

This repository follows **Optimistic Mainline Governance** by reference, not by copying the doctrine here.

Small direct commits to `main` are acceptable when explicitly authorized, scoped, reversible, inspectable by diff, and reported after completion.

Because Mare Nostrum concerns strategic, political, institutional and Mediterranean cooperation frames, use an issue, checkpoint, branch, PR, staged patch or human validation when a change could convert a scenario into doctrine, imply a partnership, create an institutional claim, alter a legal/political position, or affect several repositories.

## Editorial discipline

When drafting documents:

- prefer structured Markdown;
- use explicit headings;
- separate diagnosis, proposal, risks, objections, and next steps;
- keep documents self-contained enough to be evaluated without hidden context;
- preserve the distinction between source documents and derived publications;
- avoid rhetorical escalation unless the requested output is explicitly a public/polemical formulation.

For public-facing texts, distinguish:

- institutional version;
- civic version;
- campaign version;
- technical version;
- long-form source version;
- short derived publication.

## Strategic discipline

Mare Nostrum may include strong claims about sovereignty, capability, anti-capture, energy, compute, islands, and Mediterranean cooperation.

Agents must nevertheless avoid:

- inflammatory wording;
- unverifiable geopolitical claims;
- overclaiming technical feasibility;
- treating future alliances as existing alliances;
- confusing moral critique with operational strategy;
- confusing a desired consortium with an already existing institution.

When in doubt, use cautious language:

```text
This document proposes...
This hypothesis should be tested...
This remains to be verified...
This is a possible institutional path...
```

## Relation to FractaVolta

`FractaVolta` is the more operational and technical repository.

Do not duplicate technical implementation details here unless they are needed to explain the Mare Nostrum framework.

Use this repository for:

- Mediterranean framing;
- cooperation models;
- territorial capability;
- consortium logic;
- anti-capture principles;
- policy-facing documents;
- strategic synthesis.

Use `FractaVolta` for:

- technical architecture;
- energy packet systems;
- compute/inference infrastructure;
- operational design;
- implementation-oriented specifications.

## Relation to Cogentia

`Cogentia` governs method.

When working on Mare Nostrum, apply Cogentia principles:

- conversation is not yet corpus;
- trace is not yet decision;
- derived product is not source;
- stabilization requires explicit validation;
- continuations must be marked;
- objections must be preserved when relevant;
- agents suggest and prepare, but do not decide.

## Agent permissions

Agents may:

- draft documents;
- summarize existing material;
- propose structure;
- identify contradictions;
- mark continuations;
- prepare public versions from source documents;
- suggest links between repositories;
- produce objections and risk analyses.

Agents must not, without explicit human authorization:

- publish;
- send;
- sign;
- commit;
- create official institutional claims;
- state that a partnership exists;
- state that a legal status has been obtained;
- convert a scenario into doctrine;
- remove uncertainty markers;
- erase dissenting or cautionary notes.

## Required checks before stabilization

Before proposing a document as stable, check:

1. Is the document a source document or a derived product?
2. Are facts, hypotheses, interpretations, and proposals separated?
3. Are external factual claims cited or marked for verification?
4. Is the relation to Corsica, islands, Mediterranean shores, and partners precise?
5. Is anti-capture treated as an operational design problem, not only as denunciation?
6. Are short-term tactics distinguished from long-term doctrine?
7. Are open continuations explicitly listed?
8. Does the document remain compatible with Cogentia’s corpus/pipeline method?

## Local invariant

Mare Nostrum is not only a narrative of Mediterranean cooperation.

It is a test case for a broader principle:

```text
A territory becomes capable when it can transform its local resources,
constraints, knowledge, energy, institutions, and traces into transmissible,
governed, anti-capture capacities.
```

All work in this repository should preserve that invariant.
