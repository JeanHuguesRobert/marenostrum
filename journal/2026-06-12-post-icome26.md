# Journal de bord — Après ICOME’26

## 1. Contexte

ICOME’26 s’est tenu à Corte et s’est achevé le 12 juin 2026.

MareNostrum avait été proposé comme cadre transversal de clôture ou de prolongement, notamment à travers l’hypothèse :

> MareNostrum : vers une souveraineté computationnelle démocratique en Méditerranée — du photon à l’exergy token.

La proposition s’inscrivait dans le prolongement de Constellia, de FractaVolta, de la souveraineté énergétique méditerranéenne et de la souveraineté computationnelle démocratique.

Elle a été documentée publiquement, notamment via une issue GitHub et plusieurs courriers tracés dans le registre des interactions.

## 2. Observation

ICOME’26 n’a pas constitué un point d’entrée institutionnel effectif pour MareNostrum.

À ce stade :

- la session spéciale MareNostrum n’a pas été intégrée comme telle ;
- aucun relais public ou médiatique fort n’a été identifié ;
- les réponses institutionnelles ont confirmé que la proposition ne trouvait pas facilement sa place dans le cadrage existant ;
- les pistes périphériques ouvertes autour de PRIMA, de l’Université, de partenaires énergie ou de relais scientifiques n’ont pas produit de continuation structurante immédiate.

Ce constat ne doit pas être lu comme un simple échec.

Il constitue une donnée expérimentale.

## 3. Apprentissage principal

ICOME’26 confirme que MareNostrum ne peut pas dépendre d’un simple créneau dans un colloque existant.

La proposition est trop structurante pour être absorbée comme une communication supplémentaire.

Elle articule simultanément :

- énergie solaire méditerranéenne ;
- souveraineté computationnelle ;
- gouvernance démocratique ;
- infrastructure physique ;
- économie coopérative ;
- traçabilité ;
- IA ;
- souveraineté territoriale ;
- coopération méditerranéenne.

Un colloque disciplinaire ou semi-disciplinaire peut accueillir un fragment de cette proposition, mais il n’est pas nécessairement conçu pour accueillir le cadre complet.

La conclusion stratégique est donc :

> MareNostrum doit devenir son propre cadre d’accueil.

## 4. Changement de stratégie

Avant ICOME’26, le schéma implicite pouvait encore être :

```text
produire une proposition forte
→ chercher un lieu institutionnel d’accueil
→ obtenir une session ou un relais
→ constituer progressivement un réseau
```

Après ICOME’26, le schéma préférable devient :

```text
produire un corpus public
→ tracer les tentatives d’exposition
→ extraire les apprentissages
→ constituer un tableau de bord
→ identifier les manques structurels
→ créer progressivement un cadre autonome
→ inviter ensuite institutions, chercheurs, élus, financeurs et opérateurs à rejoindre ce cadre
```

Le basculement est important : il ne s’agit plus seulement de demander une place dans un cadre existant, mais de rendre visible la nécessité d’un cadre nouveau.

## 5. Formulation généralisable

Le cas ICOME’26 peut être généralisé comme un **test d’exposition institutionnelle**.

Un test d’exposition institutionnelle consiste à :

1. formuler publiquement une proposition structurée ;
2. l’adresser à une institution existante ;
3. tracer la réponse, l’absence de réponse ou le refus ;
4. distinguer l’échec de contenu de l’échec de cadrage ;
5. transformer le résultat en apprentissage ;
6. décider s’il faut adapter la proposition, changer d’interlocuteur ou créer un cadre autonome.

Dans ce cas, l’hypothèse privilégiée est un échec de cadrage, non un échec de contenu.

La proposition MareNostrum n’a pas été invalidée. Elle a rencontré un système qui ne disposait pas encore de la bonne case pour la recevoir.

## 6. Indicateurs à suivre

Les prochains indicateurs ne doivent pas seulement mesurer les réponses reçues.

Ils doivent mesurer la constitution progressive d’un champ de lisibilité :

- nombre de documents publics stabilisés ;
- nombre d’interactions tracées ;
- nombre de réponses qualifiées ;
- nombre d’objections utiles ;
- nombre de partenaires potentiels identifiés ;
- nombre de dérivés publiables produits ;
- existence ou non d’un vocabulaire partagé ;
- capacité à faire comprendre MareNostrum en moins de cinq minutes à un élu, un opérateur, un journaliste, un chercheur ou un citoyen.

## 7. Continuations possibles

Actions immédiates :

- maintenir un tableau de bord MareNostrum ;
- relier chaque tentative de contact à une trace ;
- classer les tentatives selon leur fonction : recherche, institution, énergie, finance, presse, collectivité, citoyen ;
- produire une note courte post-ICOME’26 ;
- préparer une version publique non polémique : "Après ICOME’26 : pourquoi MareNostrum doit devenir un cadre autonome".

Actions de second niveau :

- formaliser un modèle minimal de consortium ouvert ;
- identifier trois premiers partenaires possibles hors cadre universitaire strict ;
- produire un paquet d’introduction différencié selon les publics : élus, opérateurs énergie, chercheurs IA, presse, citoyens, finance territoriale.

## 8. Note méthodologique

Ce journal de bord applique la seconde méthode :

- ne pas effacer les tentatives infructueuses ;
- ne pas transformer un refus en ressentiment ;
- traiter l’échec apparent comme une donnée ;
- extraire l’apprentissage ;
- rendre la continuation explicite ;
- rendre le cas réutilisable pour d’autres situations.

MareNostrum avance donc non seulement par réussite institutionnelle, mais aussi par clarification des conditions dans lesquelles les institutions existantes ne savent pas encore recevoir le problème posé.
